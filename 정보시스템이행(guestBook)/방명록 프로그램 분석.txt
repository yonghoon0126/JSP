1. 방명록 프로그램 기능 분석
빈즈를 이용한 데이터베이스 연동 프로그래밍 과정을 살펴보고 jsp에 익숙해지기 위한 것이지만,
입력, 수정, 삭제, 조회 등 모든 기능을 포함하고 있어서 방명록 외의 다른 목저으로도
활용할 수 있다.
[프로그그램 기능 목록]
번호   기능           주요내용
------------------------------
1     방명록 리스트보기   페이지 구분 없이 최신 게시물을 맨 위로 해서 보여준다
2     입력            작성자, 이메일, 비밀번호, 내용을 입력하도록 한다. 
3     수정            방명록을 작성할 때 입력한 비밀번호로 수정한다.
4     삭제            수정 화면에서 삭제하도록 한다.

2. 방명록 화면 설계
기능을 분석한 내용을 바탕으로 필요한 화면을 정리하면, 초기 화면은 리스트화면이고
입력 양식은 별도 파일로 존재하지만, 수정, 취소는 한 화면에서 처리한다.
[프로그그램 화면 목록]
번호   기능               주요내용
------------------------------
1     방명록 리스트화면      guestbook_list.jsp
2     방명록 입력 양식      guestbook_form.jsp
3     방명록 수정/취소 화면  guestbook_edit_form.jsp

3. 방명록 프로그램 설계
방명록 프로그램은 데이터베이스와ㅏ의 모든 처리를 담당할 GuestBean 클래스가
중심이 되며, 테이블 데이터 매핑을 위한 GuestBook클래스와 연동한다.

그리고 사용자 화면과 빈즈를 연결시켜주는 컨트롤러와 오류 처리를 위한 jsp파일
등이 필요한다.
[방명록 프로그그램 구성]
번호     기능                                        파일명
------------------------------------------------------------
1      데이터베이스 insert,update, delete, select      GuestBean.java
       등을 수행하는 클래스

2      방명록 데이터 객체로, 데이블 내 특정 로우와 매핑된다     GuestBook.java

3      컨트롤러                                       guestbook_controll.jsp
                                                    action code=list, insert, eddit, update

4      오류 처리를 위한 페이지                            guestboo_error.jsp

4. 방명록 테이블 설계

5. action에 따른 처리 구조 생성
모든 요청이 컨트롤러로 집중되기 때문에 컨트롤러에서는 각각의 요청을 구분하고 처리할 수 있어야 한다.

action 파라미터의 분류
---------------------------------------------------------
action값            요청 기능                 처리 방법
----------------------------------------------------------
list             방명록 목록 보여주기         response.sendRedirect()를 이용한 단순 페이지 이동.

insert      방명록 게시물 신규 등록     GuestBean클래스의 insertDB()호출 후, 성공하면 
                                  response.sendRedirect()를 이용해 페이지를 전환한다.

edit       선택한 게시물 수정 페이지로 이동    선택된 게시물의 gb_id값을 가지고 GuestBean클래스의 getDB() 메서드 호출 후,
수정할 게시물의 데이터를 GudesBook 객체로 가지고와 사용자가 입력한 비밀번호와 데이터베이스 상의 비밀번호가 일치하는지 확인한다.
일치할 경우 request scope에 저장 후 pageContext.forward()를 이용해 수정 페이지로 이동시킨다. 
만일 비밀번호가 일치하지 않으면 이전 페이지로 되돌려 보낸다.



update    게시물 수정 처리           게시물 수정 처리 페이지에서 수정된 데이터를 데이터베이스상에서 갱신하는 부분이다.
GuestBean 클래스의 updateDB()를 이용해 처리하고 response.sendRedirect()를 이용해 목록 화면으로 보낸다.


delete    게시물 삭제 처리          게시물 수정 처리 페이지에서 삭제 요청이 들어온 경우,
GuestBean 클래스의 deleteDB()를 이용해 삭제 처리 후 게시물 목록 화면으로 이동한다.
